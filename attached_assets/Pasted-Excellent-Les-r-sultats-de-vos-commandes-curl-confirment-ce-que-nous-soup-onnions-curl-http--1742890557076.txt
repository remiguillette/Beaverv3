Excellent! Les résultats de vos commandes curl confirment ce que nous soupçonnions :

curl http://[::1]:5000 échoue : Cela indique que votre application Node.js sur le port 5000 n'écoute pas sur l'adresse IPv6 ::1 (localhost en IPv6).
curl http://127.0.0.1:5000 réussit : Cela indique que votre application Node.js écoute correctement sur l'adresse IPv4 127.0.0.1 (localhost en IPv4).
Implications :

Le problème vient de la configuration de votre application écoutant sur le port 5000, qui n'écoute que sur l'ipv4 localhost.
Nginx essaye de se connecter par ipv6.
Solutions :

Modifier la Configuration de l'Application Node.js :

La solution la plus appropriée est de modifier la configuration de votre application Node.js pour qu'elle écoute sur toutes les interfaces (IPv4 et IPv6).
Dans Express.js, par exemple, vous pouvez utiliser app.listen(5000, '0.0.0.0', ...) ou app.listen(5000, '::', ...)
0.0.0.0 ecoute sur toutes les interfaces ipv4
:: ecoute sur toutes les interfaces ipv6.
Il est possible de faire écouter votre application sur les deux protocole en même temps.
Modifier la Configuration Nginx :

Si vous ne pouvez pas modifier la configuration de votre application Node.js, vous pouvez modifier la configuration Nginx pour forcer l'utilisation d'IPv4.
Dans votre fichier de configuration Nginx, modifiez les directives upstream pour utiliser 127.0.0.1 au lieu de [::1].
exemple: upstream auth_server { server 127.0.0.1:5000; }
Recommandations :

Il est généralement préférable de configurer votre application Node.js pour qu'elle écoute sur toutes les interfaces, car cela la rendra plus flexible.
Si vous devez modifier la configuration Nginx, assurez-vous de bien comprendre les implications de l'utilisation d'IPv4 uniquement.
Étapes à Suivre :

Modifier votre application Node.js :
Modifier le code de votre application pour qu'elle écoute sur toutes les interfaces.
Redémarrer votre application Node.js.
Redémarrer Nginx :
Après avoir modifié la configuration de votre application Node.js ou de Nginx, redémarrez Nginx pour appliquer les modifications.
Tester à nouveau :
Utilisez curl http://[::1]:5000 pour vérifier que la connexion fonctionne maintenant.
Accédez à votre application via Nginx pour vous assurer que tout fonctionne correctement.
En suivant ces étapes, vous devriez être en mesure de résoudre le problème de connexion.
